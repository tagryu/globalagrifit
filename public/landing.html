01 · PROBLEM

매번 같은 질문, 매번 수작업.
바이어 상담의 첫 30분은 늘 똑같다.

FOUNDER NOTE · 전북통상닥터

“전북통상닥터로 활동하면서 여러 농기계 회사의 수출을 지원해주고 있는데, 잠재적 바이어와의 상담을 하다보면 매번 처음 확인하는 것이 토양의 종류와 경사도, 그리고 농작물에 대한 정보를 파악하고 해당 정보가 판매하고자 하는 제품 사양에 맞는지 확인하는 일이었다. 이런 Pin Point를 디지털로 극복하고자 한다.”

📋
반복적인 수작업

상담마다 토양·경사·기후·작물을 일일이 검색하고 엑셀에 정리. 담당자가 바뀌면 노하우가 휘발된다.

🌐
분산된 데이터

SoilGrids, WorldClim, FAO GAEZ는 따로따로 존재. 농기구 영업 담당자가 직접 통합·해석할 수 없다.

"어디에 팔면 되는가"에 대한 시각적 답이 없으니, 신제품 출시 전 시장 사이징도 직관에 의존.

CORE MATCHING — Supabase RPC
create function match_equipment(p_max_slope, p_soils, p_climates, p_min_temp, p_max_temp)
returns table (id, name, country, geometry, fit_score, area_ha) as $$
  select
    r.id, r.name, r.country,
    st_asgeojson(r.geom)::json as geometry,
    (
      (case when r.soil_type = any(p_soils) then 1 else 0 end) +
      (case when r.climate_zone = any(p_climates) then 1 else 0 end) +
      (case when r.avg_slope_deg <= p_max_slope then 1 else 0 end) +
      (case when r.avg_temp_c between p_min_temp and p_max_temp then 1 else 0 end)
    )::numeric / 4 as fit_score,  -- 4개 차원 가중 평균
    r.area_ha
  from regions r
  where r.avg_slope_deg <= p_max_slope;  -- 하드 컷오프
$$;
